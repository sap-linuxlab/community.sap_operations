# sap_monitor_nw_response Ansible Module

## Description

This Ansible Module returns response time metrics of SAP Netweaver instance using `sapcontrol` function `GetAlertTree`.

## Parameters

| Parameter | Required | Default | Details |
| --- | --- | --- | --- |
| nw_sid | Yes | | SAP Netweaver SID |
| nw_instance_number | Yes | | SAP Netweaver Instance Number |


## Returned Values
Each returned parameter can be retrieved from registered variable.

| Parameter | Details |
| --- | --- |
| database_response_time | Database Response Time parsed from `DBRequestTime` |
| dialog_response_time | Dialog Response Time parsed from `ResponseTimeDialog` |
| frontend_response_time | Frontend Response Time parsed from `FrontendResponseTime` |
| number_users | Current logged in users parsed from `UsersLoggedIn` |

**NOTE:**

- Values `-` are generated by `sapcontrol` when instance is unavailable or metric is not captured.

### Example
Example of getting response time metrics of SAP Netweaver with SID `B01` and Instance Number `01` of Type `PAS`. 

```yaml
---
- name: Ansible Play
  hosts: bw
  become: true

  tasks:

    - name: Execute Ansible Module sap_monitor_nw_response
      community.sap_operations.sap_monitor_nw_response:
        nw_sid: B01
        nw_instance_number: '01'
      register: __sap_monitor_nw_response

    - name: Show returned values from sap_monitor_nw_response
      ansible.builtin.debug:
        var: __sap_monitor_nw_response
```

Example of returned values.
```console
ok: [b01hana] =>
    __sap_monitor_nw_response:
        changed: false
        database_response_time: '922'
        dialog_response_time: '-'
        failed: false
        frontend_response_time: '1480'
        msg: SAP Check Successful
        number_users: ''
```
